# S0.2.5 — Pre-commit + commit-message lint

## Goal
Local guardrails: staged files are linted/formatted on commit, and commit messages
must follow Conventional Commits (matches the project's commit discipline).

## Files
- `package.json` devDeps + `prepare` script (husky or simple-git-hooks)
- `.husky/pre-commit` (lint-staged) and `.husky/commit-msg` (commitlint) — or equivalents
- `lint-staged` config
- `commitlint.config.js`

## Map slice
None. Commit discipline per `CLAUDE.md` (Project profile).

## Acceptance criteria
- [ ] Committing a file with a lint error is blocked locally.
- [ ] A non-conventional commit message (e.g. "stuff") is rejected.
- [ ] A conventional message (e.g. `feat(core): ...`) passes.
- [ ] **Windows note:** hooks run via Git's bundled bash — verify they execute on this machine (see memory `vulyk-windows-hooks`).

## Verification
`git commit -m "bad" --dry-run` style check is non-trivial; document the manual test in implementation notes and ensure `pnpm lint-staged` runs clean.
