# S0.2.1 — pnpm workspace + package skeletons

## Goal
After this story the repo is a working pnpm monorepo: `pnpm i` succeeds and the five
`@arch/*` packages exist as empty-but-typechecking TypeScript packages that can import
each other per the dependency rules in ADR-0003.

## Files
- `package.json` (root, `"private": true`, workspaces scripts), `pnpm-workspace.yaml`
- `tsconfig.base.json` (strict) + per-package `tsconfig.json`
- `packages/core/package.json` + `packages/core/src/index.ts`
- `packages/protocol/package.json` + `packages/protocol/src/index.ts`
- `packages/server/package.json` + `packages/server/src/index.ts`
- `packages/client/package.json` + `packages/client/src/index.ts`
- `packages/bots/package.json` + `packages/bots/src/index.ts`
- `.npmrc` (if needed), `.nvmrc` (Node 22)

## Map slice
None yet (greenfield). Respect package boundaries from `docs/specs/roadmap/ROADMAP.md` §2 and ADR-0003.

## Acceptance criteria
- [ ] `pnpm i` completes with no errors.
- [ ] `@arch/core` has **zero runtime dependencies** (ADR-0003 invariant).
- [ ] `@arch/protocol`/`server`/`client`/`bots` can import `@arch/core` types; `@arch/core` imports none of them.
- [ ] `pnpm -r exec tsc --noEmit` passes on the skeletons.
- [ ] Package names are exactly `@arch/core|protocol|server|client|bots`.

## Verification
`pnpm i && pnpm -r exec tsc --noEmit`

## Implementation notes
- Created root `package.json` (private, `packageManager: pnpm@10.29.1`, `engines: node>=22`) + `pnpm-workspace.yaml`, `tsconfig.base.json` (strict, `module: ESNext`, `moduleResolution: Bundler`), `.nvmrc` (22), `.npmrc`.
- Created all five packages under `packages/{core,protocol,server,client,bots}/` with `package.json` (name `@arch/<name>`), `tsconfig.json` extending base, and `src/index.ts` exporting a version const placeholder.
- `@arch/core` has no `dependencies` field (only `devDependencies: typescript`); protocol/server/client each depend on `@arch/core workspace:*`; server/client also depend on `@arch/protocol workspace:*`; bots depends on `@arch/core workspace:*`.
- `pnpm i` resolved TypeScript 5.9.3; `pnpm -r exec tsc --noEmit` passed with zero errors across all five packages.
