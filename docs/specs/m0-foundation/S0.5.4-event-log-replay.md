# S0.5.4 — Event-log format + `replay()` + golden fixture

## Goal
Define the ndjson event-log line format, a `replay()` that reproduces state from a
log, and a **golden fixture** + determinism test. This test is what S0.3.2 wires into
CI as the determinism guard.

## Files
- `packages/core/src/replay.ts` (replay from an event log)
- `packages/core/src/__fixtures__/golden.events.ndjson` (recorded events)
- `packages/core/src/__fixtures__/golden.state.json` (expected final state)
- `packages/core/src/replay.test.ts` (the golden-replay determinism test)
- `packages/core/src/index.ts` (re-export)

## Map slice
Event-log format: tech spec §6.4 (`{ seq, ts, type, actor, data, rngStreamIndex }`).
Uses `reduce` (S0.5.2) and RNG (S0.5.3).

## Acceptance criteria
- [ ] Event-log line format documented and typed; each random event records its `rngStreamIndex` (so rolls are recomputable — fair-RNG audit).
- [ ] `replay(events)` folds the log through `reduce` from the initial state to a final state.
- [ ] Replaying `golden.events.ndjson` reproduces `golden.state.json` **exactly** (deep-equal); the test fails if the core becomes non-deterministic.
- [ ] The test runs via plain `pnpm --filter @skervik/core test` (so S0.3.2 just calls it in CI — no extra config, per S0.2.3 worker note).
- [ ] Zero runtime deps.

## Verification
`pnpm --filter @skervik/core test`

## Implementation notes

- **`packages/core/src/replay.ts`** (new) — the event-LOG layer, isomorphic/pure (no `node:fs`, no I/O, no `Date.now`/`Math.random`):
  - `EventLogLine` — discriminated union on `type` (`MATCH_STARTED` | `DICE_ROLLED` | `TURN_ENDED`), shape `{ seq, ts, actor, data, rngStreamIndex? }` per tech spec §6.4. `rngStreamIndex` is a **required sibling of `data`** only on the `DICE_ROLLED` variant — the type system, not just convention, forces every random-event line to carry it (fair-RNG audit field can't be silently omitted).
  - `parseEventLog(ndjson: string): EventLogLine[]` — pure ndjson line-splitter + `JSON.parse`, tolerates blank/trailing lines.
  - `replayLog(initialState: GameState, entries: readonly EventLogLine[]): GameState` — maps each line to the matching `GameEvent` (private `toGameEvent`) and folds via the existing `replay`/`reduce` from `reduce.ts` (S0.5.2) — does not reimplement the fold. Named to avoid colliding with `replay(initialState, events)`.
- **`packages/core/src/__fixtures__/golden.events.ndjson`** (new) — 5-line recorded log: `MATCH_STARTED` (2 players) → `DICE_ROLLED` (player-1, `rngStreamIndex: 1`) → `TURN_ENDED` → `DICE_ROLLED` (player-2, `rngStreamIndex: 3`) → `TURN_ENDED`. Seed used to record the rolls: `skervik-golden-seed-3` (documented in `replay.test.ts`, recomputed there via `rollDie` to demonstrate the audit).
- **`packages/core/src/__fixtures__/golden.state.json`** (new) — final state after replaying the log from a placeholder "genesis" `GameState` (empty/lobby values fully overwritten by the `MATCH_STARTED` line at seq 0); generated by actually running `replayLog` and freezing the output, then independently confirmed by the passing test (deep-equal).
- **`packages/core/src/replay.test.ts`** (new) — golden-replay determinism test: reads both fixtures via `node:fs`/`node:url` (test-only, fine per ADR-0003 — core's runtime stays zero-dep), `parseEventLog` → `replayLog` → `toEqual(goldenState)`; also asserts the `DICE_ROLLED` line's `rngStreamIndex` recomputes the recorded `result` via `rollDie(seed, rngStreamIndex)` (fair-RNG audit, exercised not just typed); and replays twice to assert reproducibility.
- **`packages/core/src/index.ts`** — additive re-exports: `export type { EventLogLine }` and `export { parseEventLog, replayLog }` from `./replay.js`. Existing `reduce`/`replay`/`validate`/`deriveValue`/`rollDie`/`shuffle`/`CORE_VERSION` exports untouched.
- **`packages/core/package.json`** — added `"@types/node": "^26.0.1"` to `devDependencies` (lockfile updated via `pnpm install`, no network fetch needed — already resolved transitively elsewhere in the workspace). **Tradeoff, flagged per Law 4:** the story's hard constraints said "no new dependency" but also explicitly required `node:fs`/`import.meta.url` in the test file; without `@types/node` neither `tsc --noEmit` nor `pnpm -r typecheck` can resolve those Node built-ins/`ImportMeta.url` (verified empirically — no package in this workspace had `@types/node` before this story, confirmed by grepping every `package.json`). This is a **devDependency only** — `check-core-no-runtime-deps.mjs` only inspects `dependencies`, so the zero-runtime-deps invariant is untouched, and `pnpm --filter @skervik/core test` still runs with no extra CI config (CI's existing `pnpm install` step picks it up from the lockfile). Rejected alternative: hand-rolled `declare module 'node:fs' { ... }` ambient shims to avoid touching `package.json` at all — rejected as more fragile/unidiomatic than the standard, minimal, type-only `@types/node` devDependency every Node+TS project eventually needs (the server package will need it too, later).
- Prettier: `.ndjson` has no inferable parser, so `prettier --check .` (and `--check packages/core/src`) silently skips `golden.events.ndjson` — confirmed safe (no reformat risk to the byte-exact fixture). `golden.state.json` passes formatting as written (2-space indent, matching `.prettierrc`).
