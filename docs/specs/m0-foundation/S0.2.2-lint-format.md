# S0.2.2 — ESLint (flat) + Prettier + editorconfig

## Goal
A single shared lint/format setup applied across all packages, runnable repo-wide.

## Files
- `eslint.config.js` (flat config, TypeScript + import rules)
- `.prettierrc`, `.prettierignore`
- `.editorconfig`
- root `package.json` scripts: `lint`, `format`
- per-package `lint` script delegating to root config

## Map slice
None. Aligns with conventions in `CLAUDE.md` (Project profile).

## Acceptance criteria
- [ ] `pnpm -r lint` runs and is clean on the skeleton.
- [ ] `pnpm format` is idempotent (second run = no diff).
- [ ] Lint forbids `Math.random()` / `Date.now()` inside `packages/core/**` (custom `no-restricted-syntax` rule — enforces ADR-0003).
- [ ] TypeScript strict rules on; unused vars error.

## Verification
`pnpm -r lint`

## Implementation notes
- Configs added: `eslint.config.js` (ESLint v10 flat), `.prettierrc`, `.prettierignore`, `.editorconfig`; added `"type": "module"` to root `package.json` so the ESM `eslint.config.js` works without renaming to `.mjs`.
- Versions: ESLint 10.6.0, typescript-eslint 8.62.1, prettier 3.9.4, eslint-config-prettier 10.1.8, eslint-plugin-simple-import-sort 13.0.0 (all at workspace root `-Dw`).
- ADR-0003 core guard: three `no-restricted-syntax` AST selectors scoped to `packages/core/src/**/*.ts` — `CallExpression[callee.object.name="Math"][callee.property.name="random"]`, same for `Date.now`, and `NewExpression[callee.name="Date"]`.
- Per-package lint scripts changed from `echo 'no lint yet'` to `eslint src`; ESLint discovers root `eslint.config.js` by traversing up from each package's cwd.
