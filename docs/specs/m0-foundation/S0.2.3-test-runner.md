# S0.2.3 — Vitest per package + smoke tests

## Goal
A working test runner in every package, with one trivial passing test each, runnable
repo-wide. Establishes the harness the core determinism test (S0.3.2) will plug into.

## Files
- root `vitest.config.ts` (or per-package configs + workspace)
- `packages/*/src/*.test.ts` (one smoke test per package)
- root `package.json` script: `test` → `vitest run` recursively
- per-package `test` scripts

## Map slice
None.

## Acceptance criteria
- [ ] `pnpm -r test` runs all packages and is green.
- [ ] `pnpm --filter @skervik/core test` works standalone (core is testable in isolation — ADR-0003).
- [ ] Coverage reporter configured (text + lcov) even if thresholds are 0 for now.
- [ ] CI can call the same command (no interactive watch).

## Verification
`pnpm -r test`

## Implementation notes
Vitest v4.1.9 installed as workspace-root devDep alongside `@vitest/coverage-v8`. Chose per-package `vitest.config.ts` (not workspace config) — each package runs `vitest run` independently, satisfying the ADR-0003 core-isolation requirement without coupling packages together. Coverage wired with v8 provider, `text` + `lcov` reporters, all thresholds at 0. Smoke tests import directly from `./index.js` (ESM relative) so no build artefacts are needed.
