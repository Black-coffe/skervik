# S0.2.4 — Build pipeline (libs + client)

## Goal
Every package builds: libraries (`core`/`protocol`/`bots`) emit dual ESM/types via
tsup (or tsc), `server` compiles, and `client` builds + dev-serves via Vite (with the
PWA plugin stubbed). Path aliases resolve `@arch/*` in dev and build.

## Files
- `packages/{core,protocol,bots}/tsup.config.ts` (or tsc build config)
- `packages/server/` build config
- `packages/client/vite.config.ts` (React + PWA plugin + `@arch/*` aliases)
- root `package.json` script: `build` → recursive build
- `tsconfig` `paths` for `@arch/*`

## Map slice
None. Stack per ADR-0002 (Pixi/React/Vite) and ADR-0004 (Node/Fastify/Colyseus).

## Acceptance criteria
- [ ] `pnpm -r build` is green; libs emit `dist/` with `.js` + `.d.ts`.
- [ ] `pnpm --filter @arch/client dev` serves a page on localhost.
- [ ] `pnpm --filter @arch/client build` produces a deployable static bundle.
- [ ] Aliases work in both dev (Vite) and typecheck (tsc).

## Verification
`pnpm -r build`
